# 统一订单业务对象字段定义

## 文档说明

本文档整合了三个业务对象的字段定义：
1. **零售订单**（ord_sal_order_t / ord_sal_order_item_t）
2. **电商订单**（x3_ec_order_core / x3_ec_order_core_sub / x3_ec_order_core_sub_ext）- 共202个字段
3. **会员积分订单**（t_mall_order / t_mall_order_line）- 共106个字段（订单表59个字段，订单行表47个字段）

通过业务语义分析，将相同业务含义的字段整合为统一字段名，便于跨业务对象的数据处理和分析。

---

## 统一业务对象命名

**统一业务对象名称**：统一订单业务对象（Unified Order Business Object）

## 字段整合说明

### 整合原则

1. **业务语义相同**：将业务含义相同或相似的字段整合为统一字段名
2. **保留差异**：对于业务含义不同但字段名相似的字段，分别列出
3. **空值处理**：如果某个业务对象没有对应字段，在表格中标记为 "-"
4. **数据类型映射**：统一字段的数据类型以最通用的类型为准，具体实现时根据业务对象进行转换

### 字段映射规则

- **订单主键ID**：零售订单使用 `orderId`，电商订单使用 `system_id`，会员积分订单使用 `id`
- **订单业务编号**：零售订单使用 `orderNo`，电商订单使用 `platform_bills_id`，会员积分订单使用 `order_id`
- **订单状态**：三个业务对象都有状态字段，但状态值定义不同，需要统一映射
- **价格金额**：零售订单使用 `BigDecimal`，电商订单使用 `Text`，会员积分订单使用 `decimal`，需要统一处理
- **时间字段**：零售订单使用 `Date`，电商订单使用 `Date`，会员积分订单使用 `datetime`，需要统一格式

## 使用建议

1. **数据转换**：在使用统一业务对象时，需要根据来源业务对象进行数据类型和格式转换
2. **字段映射**：建立字段映射表，将统一字段名映射到各业务对象的具体字段
3. **业务规则**：不同业务对象的相同字段可能有不同的业务规则，需要根据业务对象类型进行区分处理
4. **扩展性**：统一业务对象设计时考虑未来可能新增的业务对象类型
